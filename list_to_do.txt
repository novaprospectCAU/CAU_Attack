1. 게임 플레이 내부 세부사항
- 그래픽 구현 방식은 미정, 단 일단은 OpenGL을 통한 레이캐스팅 방식.
- 무기는 교체가 가능해야 하며, 최소 3개 구현 예정 (현 계획 : 1. 일반 자동소총 , 2. 산탄총, 3. 저격총)
- 줌인은 3번(저격총)에 한해서만 구현할 예정 (확정은 아님)
- 3번 무기의 줌아웃 상태를 제외한 모든 경우(1,2번의 일반모드, 3번의 줌인 모드) 크로스헤드가 존재
- HUD로 체력, 현재 들고 있는 무기의 그림 및 번호와 이름, 남은 탄약 수
- 피격시 화면 흔들림 구현X 대신 화면이 붉어지는 효과 구현 예정
- 키보드 : 이동, 무기 변경, 재장전 가능
- 마우스 : 타격, 줌인-줌아웃(3번 무기 한정)
- 앉기,달리기,눕기 등의 일반적인 FPS게임에서의 모션은 구현 미정
- 반동 존재 : Random()으로 구현 - CS:GO스타일로 리코일 시스템 예정
- 사망시 : 쓰러지는 애니메이션 구현 미정, 플레이어는 메시지와 함께 일정 시간 이후 자동 리스폰
- 배경 및 플레이어, 모든 스프라이트는 둠1 스타일로 제작 예상
- 이동 스타일은 WASD + 마우스(하프라이프1 스타일)
- 이동가속도 구현 예정
- 타격 부위별 데미지 차등 적용(비곤 스타일 : 머리-치명상 / 팔다리복부-중상 / 다리-경상)
- 무기의 구매는 향후 구현할 예정

2. 네트워크 관련 세부사항
- 최대 인원 : 미정 , 현재로는 1 VS 1 먼저 구현할 예정
- 방향은 멀티플레이가 가능해야 하며, 킬로그, HUD, 메인메뉴와 서버메뉴, 플레이 화면이 분리되어야 한다.
- 핑 : 100이상 혹은 0이하면 자동으로 서버에서 퇴출, 메인메뉴로 이동하며 에러메시지를 클라이언트에 전송한다.
- 네트워크 구현 방식은 미정, 단 일단은 UNIX 기반 네트워킹을 선호하며, 경우에 따라 TCP/IP 소통방식에 변화 가능하다.


3. 서버 내부 세부사항
- 게임 데이터는 저장하지 않는다. - 개인 데이터는 사용자 이름만을 받으며, 종료 후 어디에도 저장하지 않는다.
- 서버는 C, 소켓 프로그래밍을 통해 구현할 예정, 상황에 따라 C++로 이전 가능

4. 실행 단계 세부사항
- 먼저 메뉴창으로 연결한다.
- 게임 서버 선택 / 설정 / 종료 탭 생성
- 게임 서버 선택은 열려있는 서버 창을 고를 수 있는 탭
- 설정에서는 마우스 이동 속도, 소리 크기,
